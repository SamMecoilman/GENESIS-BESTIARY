---

## 1. コア体験の定義

**「仲間と協力してモンスターを狩り、素材で装備を作る達成感」**

これがゲームの核。すべての実装はこの体験を強化するために存在する。

---

## 2. 絶対に実装しない機能

| 機能 | 理由 |
|------|------|
| PvP（対戦） | 協力がコア体験 |
| 自動戦闘 | プレイヤースキルが重要 |
| スタミナ制限 | 遊びたいときに遊べる |
| ガチャ（装備） | ゲーム性崩壊 |
| Pay to Win | MHF の二の舞 |

---

## 3. MHらしさの定義

以下の要素は「MHらしさ」として必須:

- 納刀/抜刀の概念
- 回復時の硬直（ガッツポーズ）
- 砥石による切れ味回復
- 剥ぎ取りによる素材入手
- クエスト制（制限時間・死亡回数）
- モンスターの怒り/疲労状態
- 咆哮によるバインド

---

## 4. 技術的制約

### プラットフォーム
- v1.0: PC限定（Steam）
- 将来: Switch / モバイル対応可能な設計

### ネットワーク
- 最大4人協力
- Host-Client方式
- v1.0: Steam Relay（無料）
- 将来: Photon移行でクロスプレイ

### アート
- ローポリ（PS2相当）
- テクスチャ: 512x512上限
- モジュラー式キャラクター

## 6. データ駆動の原則

- **ハードコード禁止**: 数値は必ずScriptableObjectか設定ファイルに
- **新コンテンツ = データ追加**: コード変更なしで追加できる設計
- **外部化優先**: JSONで調整可能にする

---

## 7. ステートマシン必須

プレイヤー、モンスター、UI、すべてにステートマシンを使用する。
if文の羅列は禁止。

```csharp
// NG
if (isAttacking) { ... }
else if (isDodging) { ... }
else if (isUsingItem) { ... }

// OK
switch (currentState)
{
    case PlayerState.Attack: ...
    case PlayerState.Dodge: ...
    case PlayerState.UseItem: ...
}
```

---

## 8. Phase順守

実装は必ず以下の順番で行う:

1. **Phase 1**: 縦切りコア（プレイヤー、モンスター1体、1マップ）
2. **Phase 2**: MH要素追加（回復硬直、納刀、砥石、怒り等）
3. **Phase 3**: 拡張（部位破壊、罠、環境要素）
4. **Phase 4**: ネットワーク（Mirror導入、同期）

Phaseをスキップしない。前のPhaseが完了してから次へ。

## 11. 収益モデル

### 基本方針
- **買い切り型**（月額なし）
- **課金は見た目のみ**（強さに影響しない）
- **全プレイヤーが対等**に遊べる

### 本体価格
- 1,500〜2,500円（インディー価格）

### 課金アイテム（見た目のみ）
- 重ね着（防具の見た目変更）
- 髪型・顔パーツ追加
- エモート・ジェスチャー
- 武器の見た目（性能同一）
- ボイス追加

### DLC（有料アップデート）
- 追加モンスターパック
- 追加マップパック
- 追加ストーリー

### 絶対禁止（MHFの教訓）
| 禁止事項 | 理由 |
|----------|------|
| 攻撃力・防御力に影響する課金 | Pay to Win |
| 素材・金の販売 | ゲーム性崩壊 |
| ガチャ（見た目含む） | 射幸心煽り |
| スタミナ制限 | プレイ時間制限 |
| 月額・サブスク | 継続負担 |
| 課金コース（能力UP系） | 格差発生 |

### サービス継続のための設計
- 低コスト運営（Steam Relay = サーバー費用なし）
- 技術的負債回避（Unity LTS、モジュラー設計）
- 新規参入しやすさ（複雑システムは後回し）
- コミュニティ維持（課金で差がつかない）
